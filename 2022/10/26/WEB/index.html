靶场的搭建与配置

搭建服务器

1.WAMP概念
WAMP:Windows+Apache+Mysql/MariaDB+PHP/perl/python,这是一组常用来搭建动态网站或者服务器的开源软件,本身都是各自独立的软件,但是因为经常被放在一起使用,兼容度越来越高,共同组成了一个强大的Web应用程序平台。

Apache:一个软件基金会的名字。Apache HTTP Server也就是apache服务器。可以理解成电脑上的一个应用程序,作用是将电脑变成一台服务器,让电脑开放特定的网络端口,用以接受来自网络上发送到这台机器的HTTP请求,对请求的内容进行处理并做出相应的响应

Mysql:是一个关系型数据库管理系统。

PHP:超文本预处理器,是一种通用开源脚本语言,PHP是在服务器端执行的脚本语言。
2.phpstudy安装
phpstudy:是一个PHP调试环境的程序集成包(Apache+mysql+php)

web漏洞靶场

靶场:本来是指提供了一个一个靶标(目标)供射手练习枪法的地方,但本课程所说的靶场并不是现实
中的靶场,而是在计算机中集成了一个个安全漏洞的题目集,也叫做靶场。
靶场可以随意的被攻击,即使被破坏了也不会对真实的环境有任何的影响,可还原性强。
DVWA:是用PHP+Mysql编写的一套用于常见WEB漏洞教学和练习的程序。包含了SQL注入、XSS、盲
注等常见的一些安全漏洞。

Docker

1.安装docker
Docker :是一个开源的应用容器引擎,让开发者可以打包他们的应用以及依赖包到一个可移植的镜像
中,然后发布到任何流行的 Linux或Windows操作系统的机器上,也可以实现虚拟化。容器是完全使用
沙箱机制,相互之间不会有任何接口。
Docker是网络安全中必学不可的一个容器工具,在其中,我们能够快速创建、运行、测试以及部署应用
程序。我们对一些漏洞进行本地复现时,可以使用Docker快速搭建漏洞环境,完成复现学习。
2.使用docker运行漏洞靶场

基础工具的安装使用

Linux基础

1.Linux目录结构
/:
根目录,每一个文件和目录都从这里开始。只有root用户具有该目录下的写权限。
/bin:
用户二进制文件,包含二进制可执行文件。系统的所有用户使用的命令都设在这里,例如:ps,ls,ping,
grep,cp等。
/boot:
这里存放的是启动Linux时使用的一些核心文件,包括一些连接文件以及镜像文件。
/dev:
设备文件,dev是Device(设备)的缩写, 该目录下存放的是Linux的外部设备,在Linux中访问设备的方式
和访问文件的方式是相同的。
/etc:
这个目录用来存放所有的系统管理所需要的配置文件和子目录。
/home:
用户的主目录,在Linux中,每个用户都有一个自己的目录,一般该目录名是以用户的账号命名的。
/lib:
这个目录里存放着系统最基本的动态连接共享库,其作用类似于Windows里的DLL文件。几乎所有的应用程序
都需要用到这些共享库。
/opt:
可选的附加应用程序,这是给主机额外安装软件所摆放的目录。
/root:
该目录为系统管理员,也称作超级权限者的用户主目录。
/sbin:
s就是Super User的意思,系统二进制文件,在这个目录下的linux命令通常由系统管理员使用。
/usr:
这是一个非常重要的目录,用户的很多应用程序和文件都放在这个目录下,类似于windows下的program
files目录。
/usr/bin:
系统用户使用的应用程序。
/usr/sbin:
超级用户使用的比较高级的管理程序和系统守护程序。
/tmp:
这个目录是用来存放一些临时文件的。
/var:
这个目录中存放着在不断扩充着的东西,我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文
件。

跨站脚本攻击

前端基础

1.认识前端
跨站脚本攻击:(简称XSS)是指往WEB页面里插入恶意代码,当其他用户浏览该页时,插入到WEB页面的JS代码就会被执行,从而达到恶意攻击用户的目的。

前端:是网站前台部分,是运行在主机浏览器软件上的网页。前端技术一般分为前端设计和前端开发。前端设计是指视觉设计,前端开发是网站的前台代码实现,包括HTML,CSS,JS等技术。

JavaScript:是由ECMAScript(描述了该语言的语法和基本对象),文档对象模型(描述处理网页内容的方法和接口),浏览器对象模型(描述与浏览器进行交互的方法和接口)

特点:是一种解释性脚本语言,主要用来向HTML页面添加交互行为,可以直接嵌入HTML页面,但写成单独的JS文件有利于结构和行为的分离

2.HTML
是超文本标记语言,标签语言是一种将文本以及文本相关的其他信息结合起来,展现出关于文档结构和数据处理细节的电脑文字编码。

常见标签:
body:主体部分
script:用于定义客户端脚本,比如JS,既可以包含脚本语句,也可以通过src属性指向外部脚本文件
a:超链接标签。
img:图片标签。

标签格式
1.标签名+文本+标签名
2.标签名 属性名=属性值+文本+标签名

事件处理程序

事件:是用户或浏览器执行的某种动作。浏览器可以除法页面加载和卸载事件。

事件处理程序:就是响应事件而调用的函数。当事件发生时,可以使用JS事件处理程序(事件监听器)来检查它们并执行特定的任务或一组任务。

BOM介绍

浏览器对象模型,提供了访问窗口对象的方法,可以用来弹出对话框。

BOM操作:让浏览器弹窗
alert():警告弹窗
comfirm():确定弹窗
prompt():提示弹窗

获取Cookie信息:document.cookie

DOM介绍

文档对象模型,可以使程序和脚本能够动态访问和更新文档的内容、结构及样式。
实际上就是将文档转换为对象。在网站页面中有需要子页面的元素,当页面到达浏览器时浏览器会为页面创建一个顶级的Document object文档对象,接着生成各个子文档对象,每个页面元素对应一个文档对象,每个文档对象包含属性、方法和事件。可以通过JS脚本队文档对象进行编辑,从而修改页面的元素。

跨站脚本攻击漏洞介绍

1.介绍
成因:动态网页的Web网页对用户提交请求参数未做充分的检查过滤,在提交的数据中掺入了HTML代码,并且未加编码地输出到用户的浏览器,攻击者恶意提交的payload会被受害用户的浏览器解释执行。

Payload
中文含义是有效载荷,在XSS中指代攻击代码或攻击语句。

类型
1.反射型:Payload经过后端,不经过数据库。
2.存储型:Payload经过后端,经过数据库。
